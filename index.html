Why MNemo

The SQLite of agent memory

Every other memory solution is cloud-hosted, tied to a heavyweight database, or a raw vector store with no agent-native abstractions. MNemo fills that gap.

рҹ“„

Single file

The entire memory store вҖ” vectors, metadata, indexes, WAL вҖ” lives in one .mnemo file. Copy it, back it up, email it, delete it.

рҹ”җ

Encrypted at rest

AES-256-GCM page-level encryption with Argon2id key derivation. Two-tier key hierarchy means rekey never rewrites bulk data.

рҹ§ 

Agent-native memory

Episodic, semantic, procedural, and working memory types with multi-signal recall вҖ” similarity blended with recency, importance, and frequency.

вҡЎ

Sub-linear search

IVF + Product Quantization index for sub-linear recall at scale. Exact brute-force always available as ground truth.

рҹ’ҫ

Crash-safe WAL

Write-ahead log with single-fsync commit. Kill the process mid-write вҖ” the database recovers cleanly on next open.

вҸӘ

Point-in-time recovery

Every flush creates a snapshot. Roll back to any past transaction вҖ” restores are themselves crash-safe and reversible.

Quick Start

Ten lines to persistent memory

use mnemo::{Mnemo, MnemoConfig, Memory, MemoryType, RecallRequest};

fn main() -> mnemo::Result<()> {
    let cfg = MnemoConfig { dimensions: 768, ..Default::default() };
    let mut db = Mnemo::create("agent.mnemo", "correct horse battery", cfg)?;

    db.remember(
        Memory::new("the user prefers dark mode", MemoryType::Semantic, vec![0.1, 0.2, 0.9])
            .with_agent("assistant-1")
            .with_importance(0.8),
    )?;
    db.flush()?; // durable + crash-safe

    for hit in db.recall(&RecallRequest::new(vec![0.1, 0.2, 0.9]).top_k(5))? {
        println!("{:.3}  {}", hit.score, hit.memory.content);
    }
    Ok(())
}

Agent Memory Model

Four types of memory

Structured memory types mirror how agents actually think вҖ” not just a bag of embeddings.

рҹ“–

Episodic

What happened вҖ” conversation turns, events, interactions. Consolidated from working memory at session close.

рҹ—ӮпёҸ

Semantic

What is known вҖ” facts, entities, relationships, beliefs. Long-lived knowledge with contradiction detection.

вҡҷпёҸ

Procedural

What to do вҖ” user preferences, behavioral patterns, rules. High-importance memories that guide agent behavior.

рҹ’ӯ

Working

Right now вҖ” current session context, scratchpad. Ephemeral until promoted by session consolidation.
